Executive Summary

An audit of the Properties & Maps module was performed to identify quality defects, architectural issues, and redundant modules. The module houses core functionality for property listings, Excel/CSV property data imports, and Google Maps coordinate resolution used to run driving matrix computations for clean check-in schedulers.

The review highlighted a **major code redundancy** in the Google Maps resolver module, where logic from src/lib/maps.ts was fully duplicated in the API route. Additionally, multiple instances of synchronous state setting inside React hooks (useEffect) in client-side files are present, which forces heavy cascading render passes. In the API routing, the route matrix API utilizes a public OSRM server without rate limits or fallback provisions, creating a clear production risk.

Critical & Non-Working Risks

⚠️ Cascading State Updates via Synchronous Effects (Performance Leak)
In client pages page.tsx and [id]/page.tsx, state variables are updated synchronously inside useEffect bodies during initialization. This causes Next.js to trigger a second layout rendering cycle immediately after mounting, hurting application paint times.
src/app/dashboard/properties/[id]/page.tsx (Line 585) React Antipattern 
useEffect(() => {
  try {
    const saved = localStorage.getItem('hhs_custom_property_columns');
    if (saved) setCustomColumns(JSON.parse(saved)); // Synchronous state trigger in effect
  } catch (e) {
    console.error('Failed to load custom columns', e);
  }
}, []);

Impact: Renders visual flash effects and compromises overall rendering speeds.
Resolution: Initialize state lazily instead: const [customColumns, setCustomColumns] = useState(() => JSON.parse(localStorage.getItem('hhs_custom_property_columns') || '[]')).

⚠️ Direct Dependence on Public Demo OSRM Server
The distance matrix endpoint queries Project OSRM's public demo router instance. This server is not backed by an SLA and can fail, rate limit, or experience downtimes, directly disabling driving duration calculations for listing checks.
src/app/api/route-matrix/route.ts (Line 11) API Risk 
const coordString = coordinates.map(c => `${c.lng},${c.lat}`).join(';');
const osrmUrl = `http://router.project-osrm.org/table/v1/driving/${coordString}?annotations=duration`;
const res = await fetch(osrmUrl);

Impact: High vulnerability to rate limiting (429 Status) and complete outage risk for driving time comparisons.
Resolution: Implement a fallback configuration or replace with an enterprise API (Google Maps Matrix API or a self-hosted OSRM container).
